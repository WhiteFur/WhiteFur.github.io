Wintersmith + Github Pages = Blog

Written by whiteFur

很久以前就想弄一個自己的blog，考慮了多個static site generator： Octopress，Pelican， Nikola。

Octopress不用說了，應該是目前算最多人用的，用Ruby寫成的，我沒用過。 Pelican則是Python陣營最多人用的。我有試用了一下，很容易上手，pip install Pelican就可以裝起來了。Nikola也是Python寫成的，但我在安裝的過程一直出現error，所以作罷。

最後為何選擇wintersmith呢？其實是因為我對NodeJS比較熟，一開始只有想到Ruby、Python的static site gererator，完全沒想到NodeJS應該也會有類似的東西，所以就去GOOGLE了一下，還真的有！選了一個最多人用的(github有超過1,700個star)，不過跟Octopress，Pelican比起來還是小宗。

這篇就來記錄一下wintersmith的使用方法吧！
wintersmith github repo的README其實就寫得很清楚了。

安裝

首先我們要透過npm來安裝Wintersmith, npm是NodeJS的package manager, 如同Python的pip跟Ruby的gem

  1. 安裝NodeJS
    Mac可以用homebrew安裝， 其它OS的package manager也都可以安裝node

  2. 透過npm安裝wintersmith
    $ npm install -g wintersmith

開始使用

  1. 選定一個位置建立一個blog project
    $ wintersmith new <path>

  2. 撰寫文章
    來看看剛剛建立的project
    其中contents/articles為文章內容。每篇文章為一個資料夾，各資料夾下的index.md即為該文章的markdown。wintersmith已經先幫我們建立了五篇範例。

  3. 預覽
    $ wintersmith preview 打開browser連到 http://localhost:8080/wintersmithTest就可以看到blog了

  4. build
    剛剛只是預覽而已，我們要透過$ wintersmith build產生出整個blog site的靜態檔案，wintersmith會幫我們產出在build資料夾下。

  5. deploy
    將剛剛build出來的檔案放到web server上就大功告成了！

那不想自己架web server怎麼辦呢？身為一個developer，有個Github account是很正常的，正好Github的Github Pages可以滿足我們的需求。接下來我們就來看看怎麼把website deploy到Github Pages上吧！

Github Pages

有兩種

  1. User or organization site - http://username.github.io/
    必需自己建立一個名為username.github.io的repo。

  2. Project site - http://username.github.io/repo_name/

最大的差別就只是project site的url在最後面會多加一個repo的名字。

把website local repo push到Github repo就可以透過上述的URL連到website了。
(project site必需push到gh-pages branch)

Step by Step - project site

  1. 在Github上建立repo

  2. 在一開始用wintersmith new <path>建立的資料夾下執行
    $ wintersmith build

  3. 在build資料下建立git repo 

    $ cd build
$ touch README.md
$ git init
$ git add . && git commit -m "my first blog"

  4. 建立&切換至gh-pages branch
    (如果是user or orginization site要用master) git checkout -b gh-pages

  5. push gh-pages
    $ git push origin gh-pages
